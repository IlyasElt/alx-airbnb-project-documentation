# Use Case Diagram – Airbnb Clone Backend

This directory contains the use case diagram for the Airbnb Clone backend project.  
The diagram visualizes how different actors interact with the system’s core functionalities.

## Actors
- **Guest** – Searches properties, books stays, makes payments, leaves reviews.
- **Host** – Manages property listings and views bookings.
- **Admin** – Oversees users, listings, bookings, and reviews.
- **Payment Processor** – Handles secure payment transactions.

## Main Use Cases
- User registration and authentication  
- Property listing creation and management  
- Searching and filtering properties  
- Booking and canceling reservations  
- Payment processing  
- Reviews and ratings  
- Admin-level management tools  

The diagram file `use_case_diagram.png` visually represents these interactions.
