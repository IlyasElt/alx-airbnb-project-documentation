This document lists the user stories derived from the use case diagram.  
Each user story reflects an interaction a user should be able to perform through the system.

---

## Guest User Stories

**1. User Registration**  
As a guest, I want to create an account so that I can book properties on the platform.

**2. User Login**  
As a guest, I want to log in securely so that I can access my profile and reservations.

**3. Search Properties**  
As a guest, I want to search for available properties by location, dates, and price so that I can find the best stay for my trip.

**4. Book a Property**  
As a guest, I want to make a booking so that I can reserve a property for my travel dates.

**5. Make Payment**  
As a guest, I want to pay for my booking securely so that my reservation is confirmed.

**6. Leave a Review**  
As a guest, I want to leave a review after my stay so that other users can benefit from my feedback.

---

## Host User Stories

**7. Create Property Listing**  
As a host, I want to list my property with photos and details so that guests can book it.

**8. Manage Listings**  
As a host, I want to edit or delete my property listings so that I can keep my information up to date.

**9. View Bookings**  
As a host, I want to see all reservations made for my properties so that I can prepare for incoming guests.

---

## Admin User Stories

**10. Manage Users**  
As an admin, I want to manage user accounts so that the platform remains safe and compliant.

**11. Manage Listings**  
As an admin, I want to monitor property listings so that inappropriate content can be removed.

**12. Monitor Bookings & Payments**  
As an admin, I want to view system-wide bookings and payments so that I can ensure platform integrity.
